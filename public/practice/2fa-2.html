Test Scenarios

Scenario 1: Register, login, and enable 2FA
1. Register a new account:
   - Enter a unique username
   - Enter a secure password
   - Enter a valid email
   - Click "Register"
   - Confirm the success message

2. Login to the account:
   - Enter the username you registered
   - Enter the password
   - Click "Login"
   - Verify you are logged in

3. Enable 2FA:
   - Click the "Enable 2FA" button next to your 2FA status
   - Scan the QR code with an authenticator app
   - Enter the 6-digit code from the app
   - Click "Verify and Enable 2FA"
   - Verify 2FA status changed to "Enabled"
   - Verify security level changed to "High"

4. Logout and login again:
   - Click "Logout" 
   - Login with your username and password
   - Observe that you're prompted for 2FA token
   - Enter the 6-digit code from your authenticator app
   - Verify successful login
Scenario 2: Test 2FA security bypass attempts
1. Attempt to bypass with empty token:
   - Login with username "user1" and password "test"
   - Leave the 2FA token field empty
   - Click "Login"
   - Verify login is rejected
   
2. Attempt to bypass with incorrect token:
   - Login with username "user1" and password "test"
   - Enter an invalid 2FA token (e.g., "000000")
   - Click "Login"
   - Verify login is rejected
   
3. Test session management:
   - Login successfully with username "user1", password "test", and valid 2FA token
   - Wait for 2 minutes (session duration)
   - Refresh the page
   - Verify you are logged out automatically
   
4. Test disable 2FA:
   - Login successfully with proper credentials
   - Click "Disable 2FA" under user section
   - Try submitting without a token
   - Verify it requires a valid token
   - Enter a valid token and submit
   - Verify 2FA is disabled and security level drops
